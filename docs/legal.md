# Legal And Credits

This page is informational and is not legal advice.

## Project Identity

Amethyst is a fork of TurboWarp and Scratch-related open-source projects. It is not affiliated with, endorsed by, or sponsored by Scratch, the Scratch Foundation, MIT, or TurboWarp.

Recommended public wording:

```text
Amethyst is a fork of TurboWarp inspired by Scratch-style block coding.
```

Avoid describing Amethyst as an official Scratch product.

## Scratch

Scratch is developed by the Scratch Foundation. Scratch names, marks, characters, and branding may be protected by trademark and other rights.

Amethyst should use "Scratch-style" only to describe the style of block coding, not to claim official Scratch status.

## TurboWarp

TurboWarp is an open-source project and is the main source stack Amethyst is based on.

Do not remove TurboWarp credits, license notices, or attribution required by upstream licenses.

## Three.js

Amethyst uses Three.js for 3D rendering.

Three.js and its examples/loaders are separate open-source software with their own license terms.

## Package Licenses In This Repository

Current package license declarations include:


These declarations come from the package metadata in this repository. If package metadata changes, update this page.

## What This Means Practically

- Keep upstream copyright notices.
- Keep license files and generated notices when present.
- Keep credits pages available.
- Do not hide that Amethyst is forked from TurboWarp/Scratch-related projects.
- Do not use Scratch/TurboWarp branding in a way that implies endorsement.
- Do not remove required legal text just because the product UI says Amethyst.

## Branding Rule

Product UI should say:

```text
Amethyst
```

Legal and credits pages should still mention:

```text
TurboWarp
Scratch
Scratch Foundation
MIT, where relevant to upstream notices
Three.js
Other bundled dependencies
```

## Credits Page

The app should keep a credits page or equivalent credits area. It should be reachable from the project or menu UI.

Credits should include:

- Amethyst contributors.
- TurboWarp.
- Scratch-related upstream projects.
- Three.js.
- Other major bundled open-source dependencies.

## Asset Licenses

Do not assume imported models, sounds, images, HDRIs, icons, or generated assets are free to redistribute.

For bundled assets, keep a note of:

- source
- author if known
- license
- whether redistribution is allowed
- whether attribution is required

## Contributor Reminder

When changing branding:

- Remove confusing product branding where appropriate.
- Keep required credits.
- Keep required license notices.
- Keep attribution truthful.

The goal is clean Amethyst branding, not pretending the upstream work does not exist.
